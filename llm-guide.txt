LLM USAGE GUIDE:
  Workflow: inspect -> read uids -> act (click/fill) -> inspect again.

  Quick start:
    aibrowsr --headless goto https://example.com --inspect
    aibrowsr click e3 --inspect
    aibrowsr fill e2 "hello@example.com" --inspect
    aibrowsr fill-form e2="user@test.com" e3="pass" --inspect
    aibrowsr eval "document.title"

  Key patterns:
    aibrowsr inspect                        # accessibility tree with uid=eN
    aibrowsr click e4 --inspect             # act + observe in one call
    aibrowsr fill-form e2="a" e3="b"        # batch fill
    aibrowsr eval "JSON.stringify(...)"     # complex DOM queries
    aibrowsr screenshot                     # returns file path (PNG)
    aibrowsr wait text "Success" --timeout 15
    aibrowsr type "hello"                   # type into focused element
    aibrowsr press Enter                    # press a key
    aibrowsr scroll down                    # scroll page, or: scroll e5
    aibrowsr hover e3                       # hover over element

  Multi-tab:
    aibrowsr --page tab2 goto https://other.com
    aibrowsr --page tab2 inspect

  Tips:
    - Always inspect before interacting -- uids change when the page mutates.
    - Use --inspect on click/fill to get updated state in one call.
    - Prefer inspect over screenshot (~50 vs ~100K tokens).
